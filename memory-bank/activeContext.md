## Статистика контента

**Всего статей:** Зависит от настроенных RSS-фидов

**Источник контента:** RSS-импорт из внешних источников

**Статус импорта:**
- RSS-фиды настраиваются в Payload CMS
- Автоматический импорт по cron-расписанию
- Статьи с обложками публикуются автоматически
- Статьи без обложек попадают в черновики

---

## Настроенные RSS-фиды

**Статус:** В процессе настройки

**Процесс настройки RSS-фида:**
1. Добавление URL фида в Payload CMS
2. Настройка маппинга полей (title, content, author, category, image)
3. Выбор источника автора (из фида / существующий / новый)
4. Настройка cron-расписания для автоматического импорта
5. Тестовый импорт

**Требования к RSS-фидам:**
- Наличие тега `<custom:image>` для автоматической публикации
- Корректная структура XML
- Актуальный контент

**Примеры фидов для настройки:**
- Fashion-блоги
- Новостные порталы о моде
- Бренды и дизайнеры
- Beauty-издания

## Открытые вопросы и проблемы

### Технические

**1. HTTPS не настроен**

**Статус:** Открыто  
**Приоритет:** Критический  
**Описание:** Сайт работает только по HTTP. Нужен SSL сертификат (Let's Encrypt).  
**Влияние:** SEO (Google предпочитает HTTPS), безопасность, доверие пользователей.  
**План:** Настроить Certbot + Nginx для автоматического обновления сертификата.

---

**2. RSS-фиды не настроены**

**Статус:** Открыто  
**Приоритет:** Критический  
**Описание:** Нет настроенных RSS-источников для импорта контента.  
**Влияние:** Сайт не наполняется контентом автоматически.  
**План:** Найти и настроить качественные RSS-фиды о моде и стиле.

---

**3. Sitemap не генерируется автоматически**

**Статус:** Открыто  
**Приоритет:** Средний  
**Описание:** Нет автоматической генерации sitemap.xml при публикации новых статей.  
**Влияние:** Индексация в Google может быть медленнее.  
**План:** Добавить автогенерацию sitemap в Astro.

---

**4. Нет системы internal linking**

**Статус:** Открыто  
**Приоритет:** Средний  
**Описание:** Нет автоматических "Related Articles" на страницах статей.  
**Влияние:** Слабая внутренняя перелинковка, пользователи не переходят на другие статьи.  
**План:** Добавить блок "Related Articles" на основе категории.

---

### Контентные

**5. Качество RSS-контента**

**Статус:** Открыто  
**Приоритет:** Средний  
**Описание:** Контент из RSS может быть разного качества, нужна модерация.  
**Влияние:** Риск публикации некачественного контента.  
**План:** Настроить правило: статьи без обложки попадают в черновики для ручной проверки.

---

**6. Изображения из RSS могут быть недоступны**

**Статус:** Открыто  
**Приоритет:** Низкий  
**Описание:** Изображения используются по прямым ссылкам из RSS, могут стать недоступными.  
**Влияние:** Битые изображения на сайте.  
**План:** В будущем рассмотреть загрузку изображений на Bunny CDN.

---

**7. Нет системы тегов**

**Статус:** Открыто  
**Приоритет:** Низкий  
**Описание:** Статьи имеют только категории, нет тегов для более детальной классификации.  
**Влияние:** Сложнее группировать статьи по подтемам.  
**План:** Добавить систему тегов в Payload CMS и маппинг из RSS.
