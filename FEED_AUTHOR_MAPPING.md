# Маппинг авторов в фидах

## Описание

Добавлена возможность управления авторами статей при импорте из RSS фидов.

## Вложенные пути в маппинге полей

Все поля маппинга поддерживают **вложенные пути через точку** для доступа к вложенным объектам в XML.

### Пример: извлечение изображения из media:content

Если в фиде есть структура:
```xml
<media:content url="https://example.com/image.jpg" type="image/png" expression="full" />
```

Парсер преобразует это в:
```json
{
  "media:content": {
    "@_url": "https://example.com/image.jpg",
    "@_type": "image/png",
    "@_expression": "full"
  }
}
```

Чтобы извлечь URL изображения, укажите в **"Поле изображения"**:
```
media:content.@_url
```

### Поддерживаемые поля:
- Поле заголовка
- Поле контента
- Поле описания/анонса
- Поле автора
- Поле категории
- Поле тегов
- **Поле изображения** ← особенно важно!
- Поле даты публикации

## Логика работы

При импорте статьи из фида автор определяется по следующему приоритету:

### 1. Автор из фида (высший приоритет)
Если в фиде есть поле автора (например, `dc:creator` или `author`), используется имя из этого поля:
- Если автор с таким именем существует в базе → используется существующий
- Если автора нет → создается новый автор с этим именем

### 2. Дефолтный существующий автор
Если в фиде нет поля автора, проверяется настройка "Автор по умолчанию":
- В админке можно выбрать существующего автора из списка
- Этот автор будет назначен всем статьям из этого фида, где не указан автор

### 3. Создание нового автора
Если не выбран дефолтный автор, можно указать имя для создания нового:
- В поле "Имя нового автора" вводится имя
- При первом импорте статьи без автора будет создан новый автор с этим именем
- Все последующие статьи будут использовать этого же автора

### 4. Без автора
Если ничего не настроено:
- Статья создается без автора (`author = null`)

## Настройка в админке

Откройте фид в админке CMS (`/admin/collections/feeds/{id}`):

### Секция "Маппинг полей"

1. **Поле автора** (authorField)
   - По умолчанию: `dc:creator`
   - Укажите имя поля в XML фида, откуда брать автора

2. **Откуда брать автора** (authorSource) - главное поле!
   
   **Вариант А: "Из фида (поле выше)"** (from_feed)
   - Автор берется из поля, указанного выше (например, `dc:creator`)
   - Если автор с таким именем существует - используется он
   - Если нет - создается новый автор
   
   **Вариант Б: "Существующий автор"** (existing)
   - Появляется dropdown со списком всех авторов из базы
   - Выберите нужного автора
   - Все статьи из этого фида получат этого автора
   
   **Вариант В: "Создать нового автора"** (create_new)
   - Появляется текстовое поле "Имя нового автора"
   - Укажите имя
   - При первом импорте будет создан новый автор с этим именем
   - Все статьи из этого фида получат этого автора

## Примеры использования

### Пример 1: Фид с авторами

**Настройка:**
- Поле автора: `dc:creator`
- **Откуда брать автора: "Из фида (поле выше)"**

**Результат:**
```xml
<item>
  <dc:creator>Иван Иванов</dc:creator>
  ...
</item>
```
→ Автор "Иван Иванов" (создается или используется существующий)

### Пример 2: Назначить существующего автора

**Настройка:**
- Поле автора: `dc:creator` (не важно, не будет использоваться)
- **Откуда брать автора: "Существующий автор"**
- **Выберите автора: "Редакция сайта"**

**Результат:**
Все статьи получат автора "Редакция сайта"

### Пример 3: Создать нового автора

**Настройка:**
- Поле автора: (любое)
- **Откуда брать автора: "Создать нового автора"**
- **Имя нового автора: "Автор из FashionPress"**

**Результат:**
- При первом импорте создается автор "Автор из FashionPress"
- Все статьи из этого фида получат этого автора
- При последующих импортах используется уже созданный автор

## Миграция базы данных

Добавлены новые колонки в таблицу `feeds`:
- `field_mapping_author_source` (varchar, default: 'from_feed')
- `field_mapping_existing_author_id` (integer, nullable)
- `field_mapping_new_author_name` (varchar, nullable)

Миграция применяется автоматически при запуске:
```bash
docker exec news_project-cms-1 npm run payload migrate
```

## API

Новые поля доступны через API:

```typescript
GET /api/feeds/{id}

{
  "fieldMapping": {
    "authorField": "dc:creator",
    "authorSource": "existing",  // 'from_feed' | 'existing' | 'create_new'
    "existingAuthor": 5,         // ID автора (если authorSource = 'existing')
    "newAuthorName": "Автор из FashionPress"  // (если authorSource = 'create_new')
  }
}
```

## Troubleshooting

### Авторы дублируются

Проблема: При каждом импорте создается новый автор с тем же именем.

**Решение:**
- Функция `ensureAuthor` ищет автора по имени (case-insensitive)
- Убедитесь, что имена авторов в фиде совпадают (без лишних пробелов, регистр не важен)
- Можно использовать дефолтного автора вместо создания новых

### Дефолтный автор не применяется

Проблема: Статьи создаются без автора, хотя настроен дефолтный.

**Проверьте:**
1. Поле "Поле автора" должно быть пустым или содержать несуществующее поле
2. Дефолтный автор должен быть **выбран** в dropdown
3. Перезапустите импорт после изменения настроек

### Автор создается вместо использования дефолтного

Проблема: Хотя в фиде нет автора, создается новый вместо использования дефолтного.

**Причина:**
- Скорее всего указано "Имя нового автора", но не выбран "Автор по умолчанию"
- "Имя нового автора" имеет приоритет над дефолтным

**Решение:**
- Очистите поле "Имя нового автора"
- Выберите "Автор по умолчанию" из списка
