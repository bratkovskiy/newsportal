# Изменения в логике импорта фидов

## Обзор изменений

Реализована новая логика обработки категорий и тегов при импорте статей из RSS фидов:

### Основные изменения:

1. **`<category>` из фида теперь идет в теги** (не в категории)
2. **Категория определяется специальным полем** или используется дефолтная (ID 64)
3. **Добавлен механизм маппинга полей** в админке для гибкой настройки

---

## Новая логика работы

### До изменений:
- `<category>` из фида → категория статьи
- Теги не использовались

### После изменений:
- `<category>` из фида → теги статьи
- `<custom:category>` из фида → категория статьи
- Если категория не найдена → используется категория с ID 64

---

## Маппинг полей фида

В админке для каждого фида добавлена группа полей "Маппинг полей" со следующими настройками:

| Поле | Значение по умолчанию | Описание |
|------|----------------------|----------|
| `titleField` | `title` | Поле заголовка статьи |
| `contentField` | `content:encoded` | Поле контента статьи |
| `excerptField` | `custom:secondTitle` | Поле анонса/описания |
| `authorField` | `dc:creator` | Поле автора |
| `categoryField` | `custom:category` | Поле категории (если нет → ID 64) |
| `tagsField` | `category` | Поле тегов |
| `imageField` | `custom:image` | Поле изображения |
| `dateField` | `pubDate` | Поле даты публикации |
| `defaultCategoryId` | `64` | ID категории по умолчанию |

## UI компонент предпросмотра

Добавлен новый компонент **"Предпросмотр маппинга"** в интерфейс редактирования фида.

### Возможности:
- Загрузка первого элемента из фида
- Просмотр всех доступных полей в сыром виде (JSON)
- Просмотр результата применения маппинга
- Тестирование настроек перед импортом

### Использование:
1. Создайте или откройте фид в админке
2. Сохраните фид (чтобы получить ID)
3. Нажмите "Загрузить предпросмотр"
4. Изучите доступные поля и настройте маппинг
5. Повторно загрузите предпросмотр для проверки

## API Endpoints

### Новый endpoint: `/api/feeds/:id/preview`

**Метод:** POST  
**Авторизация:** требуется (admin)

**Описание:** Загружает первый элемент из фида и применяет к нему маппинг полей.

**Ответ:**
```json
{
  "ok": true,
  "rawItem": { /* сырые данные из фида */ },
  "mapped": {
    "title": "Заголовок статьи",
    "content": "Контент...",
    "excerpt": "Анонс...",
    "author": "Автор",
    "category": "Категория или null",
    "tags": ["тег1", "тег2"],
    "image": "URL изображения",
    "date": "2024-01-01T00:00:00.000Z"
  }
}
```

## Миграция данных

### Категория с ID 64

Убедитесь, что в базе данных существует категория с ID 64, которая будет использоваться по умолчанию.

**SQL для создания (PostgreSQL):**
```sql
INSERT INTO categories (id, name) 
VALUES (64, 'Без категории')
ON CONFLICT (id) DO NOTHING;
```

### Пересоздание миграций Payload

После изменения схемы выполните:

```bash
cd apps/cms
npm run payload migrate:create
npm run payload migrate
```

## Примеры использования

### Пример 1: Стандартный RSS фид

Для стандартного RSS фида (WordPress, etc.) используйте дефолтные настройки:
- `tagsField`: `category` ✓
- `categoryField`: `custom:category` (если есть специальное поле)

### Пример 2: Кастомный фид

Если ваш фид использует другие названия полей:
- `tagsField`: `keywords` (вместо category)
- `categoryField`: `section` (вместо custom:category)
- `titleField`: `headline` (вместо title)

### Пример 3: Фид без категорий

Если фид не содержит информации о категории:
- Оставьте `categoryField` пустым
- Все статьи получат категорию с ID 64 (дефолтную)

## Тестирование

1. Создайте тестовый фид с настройками маппинга
2. Используйте "Предпросмотр маппинга" для проверки
3. Запустите импорт одного фида через "Импортировать сейчас"
4. Проверьте созданные статьи на корректность категорий и тегов

## Troubleshooting

### Проблема: Категория не создается

**Решение:** Убедитесь, что поле `categoryField` указывает на существующее поле в фиде.

### Проблема: Теги не импортируются

**Решение:** Проверьте, что `tagsField` указывает на правильное поле (обычно `category`).

### Проблема: Все статьи получают категорию 64

**Решение:** Это нормально, если в фиде нет специального поля для категории. Настройте `categoryField` или измените `defaultCategoryId`.
