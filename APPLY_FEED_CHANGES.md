# Инструкция по применению изменений в логике фидов

## Шаг 1: Создайте категорию по умолчанию (ID 64)

Если у вас еще нет категории с ID 64, создайте её вручную в админке или через SQL:

```sql
INSERT INTO categories (id, name) 
VALUES (64, 'Без категории')
ON CONFLICT (id) DO NOTHING;
```

## Шаг 2: Примените миграцию базы данных

Выполните миграцию для добавления поля `field_mapping` в таблицу `feeds`:

```bash
cd apps/cms
npm run payload migrate
```

Или вручную выполните SQL:

```sql
ALTER TABLE feeds
ADD COLUMN IF NOT EXISTS field_mapping jsonb;
```

## Шаг 3: Перезапустите CMS

```bash
cd apps/cms
npm run dev
```

## Шаг 4: Проверьте существующие фиды

1. Откройте админку CMS
2. Перейдите в раздел "Feeds"
3. Откройте любой существующий фид
4. Убедитесь, что появилась секция "Маппинг полей" с дефолтными значениями
5. Сохраните фид (не обязательно, но рекомендуется)

## Шаг 5: Протестируйте предпросмотр

1. Откройте фид в админке
2. Найдите секцию "Предпросмотр маппинга"
3. Нажмите "Загрузить предпросмотр"
4. Проверьте, что данные корректно извлекаются из фида

## Шаг 6: Настройте маппинг (если нужно)

Если ваш фид использует нестандартные поля:

1. Откройте фид
2. В секции "Маппинг полей" измените нужные поля
3. Используйте "Предпросмотр маппинга" для проверки
4. Сохраните настройки

## Шаг 7: Протестируйте импорт

1. Нажмите "Импортировать сейчас" в фиде
2. Проверьте созданные статьи:
   - Теги должны быть из поля `category` (или настроенного `tagsField`)
   - Категория должна быть из `custom:category` или ID 64
3. При необходимости скорректируйте маппинг и повторите

## Шаг 8: Обновите jobs импорта (опционально)

Если вы используете автоматический импорт через jobs:

```bash
cd apps/jobs
npm run build
# Перезапустите jobs процесс
```

## Проверка корректности

### Что изменилось:

✅ **До:** `<category>` из фида → категория статьи  
✅ **После:** `<category>` из фида → теги статьи

✅ **До:** теги не использовались  
✅ **После:** `<custom:category>` → категория, иначе ID 64

### Как проверить:

1. Создайте тестовую статью через импорт
2. Проверьте, что теги заполнены значениями из `<category>`
3. Проверьте, что категория = ID 64 или значение из `<custom:category>`

## Откат изменений (если нужно)

Если что-то пошло не так:

```bash
cd apps/cms
npm run payload migrate:down
```

Или вручную:

```sql
ALTER TABLE feeds
DROP COLUMN IF EXISTS field_mapping;
```

Затем восстановите файлы из Git:
```bash
git checkout HEAD -- apps/cms/src/rssImport.ts
git checkout HEAD -- apps/jobs/src/importFeeds.ts
git checkout HEAD -- apps/cms/src/payload.config.ts
```
