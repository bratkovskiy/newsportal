# Страницы тегов

## Реализовано

### 1. Обновлена коллекция Tags в CMS
- Добавлено поле `slug` с автогенерацией латиницей из названия
- Добавлено поле `metaTitle` (SEO) с автогенерацией "НАЗВАНИЕ ТЕГА - LADY.NEWS"
- Добавлено поле `metaDescription` (SEO) с автогенерацией "Все материалы по тегу НАЗВАНИЕ"
- Все поля доступны для редактирования в админке

### 2. Создана миграция для существующих тегов
- Файл: `/apps/cms/src/migrations/2025-12-11-01-add-tags-fields.ts`
- Миграция добавляет новые поля в таблицу `tags`
- Автоматически генерирует slug для всех существующих тегов

### 3. Создана страница списка всех тегов
- URL: `/tags/`
- Файл: `/apps/web/src/pages/tags/index.astro`
- Отображает все теги в виде облака тегов с красивым дизайном
- Каждый тег - это ссылка на страницу тега

### 4. Создана страница отдельного тега
- URL: `/tags/[slug]`
- Файл: `/apps/web/src/pages/tags/[slug].astro`
- Отображает все новости, привязанные к данному тегу
- Использует те же компоненты, что и страницы категорий (CardFeed)
- Поддерживает заглушку для битых изображений

## Запуск миграции

### В Docker окружении:
```bash
# Выполнить миграцию в контейнере CMS
docker-compose exec cms npm run migrate

# Перегенерировать типы TypeScript
docker-compose exec cms npm run generate:types

# Перезапустить контейнеры для применения изменений
docker-compose restart cms web
```

### Локально (если используется локальная разработка):
```bash
cd apps/cms
npm run migrate
npm run generate:types
```

## Структура URL

- `/tags/` - список всех тегов
- `/tags/moda` - страница тега "Мода" (slug = moda)
- `/tags/krasota` - страница тега "Красота" (slug = krasota)

## Администрирование

В админке CMS (http://localhost:3000/admin) в разделе "Tags" теперь доступны поля:
1. **Name** - название тега (обязательное, уникальное)
2. **URL (алиас)** - автоматически генерируется латиницей из названия
3. **Meta Title (SEO)** - автоматически генерируется, можно редактировать
4. **Meta Description (SEO)** - автоматически генерируется, можно редактировать

## Автогенерация slug

При создании нового тега:
- Название на русском языке автоматически транслитерируется в латиницу
- Пробелы заменяются на дефисы
- Специальные символы удаляются
- Результат используется как slug

Примеры:
- "Мода" → "moda"
- "Уличная мода" → "ulichnaya-moda"
- "Beauty & Style" → "beauty-style"

## Связь с новостями

Новости связаны с тегами через поле `tags` (relationship, hasMany).
На странице тега отображаются все опубликованные новости, которые имеют этот тег.

## Дополнительные улучшения

### Отображение тегов на странице статьи
- Теги отображаются внизу статьи после основного контента
- Каждый тег - это кликабельная ссылка на страницу тега
- Дизайн тегов соответствует общему стилю сайта

### Навигация
- Добавлена ссылка "Теги" в главное меню сайта
- Хлебные крошки на странице тега: Главная / Теги / [Название тега]
