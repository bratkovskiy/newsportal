# Проверка функциональности страниц тегов

## ✅ Что было выполнено:

1. **Обновлена коллекция Tags в CMS** с добавлением:
   - `slug` (алиас) - автогенерация латиницей
   - `metaTitle` (SEO)
   - `metaDescription` (SEO)

2. **Создана и выполнена миграция** для существующих тегов

3. **Созданы страницы:**
   - `/tags/` - список всех тегов
   - `/tags/[slug]` - страница отдельного тега с новостями

4. **Пересобраны и перезапущены контейнеры:**
   - CMS (с обновленной схемой БД)
   - Web (с новыми страницами)

## 🧪 Как проверить:

### 1. Админка CMS - Редактирование тегов
Откройте: http://localhost:3000/admin/collections/tags

**Проверьте:**
- При создании нового тега автоматически генерируется `slug` (алиас)
- Автоматически заполняются `Meta Title` и `Meta Description`
- Все поля можно редактировать вручную

**Тестовый сценарий:**
1. Создайте новый тег с названием "Уличная мода"
2. Убедитесь, что `slug` = "ulichnaya-moda"
3. Убедитесь, что `Meta Title` = "Уличная мода - LADY.NEWS"
4. Убедитесь, что `Meta Description` = "Все материалы по тегу Уличная мода"

### 2. Страница списка всех тегов
Откройте: http://localhost:4321/tags/

**Проверьте:**
- Отображаются все теги в виде облака тегов
- Каждый тег - это кликабельная ссылка
- Дизайн красивый с эффектами hover

### 3. Страница отдельного тега
Откройте: http://localhost:4321/tags/[slug] (замените [slug] на реальный slug)

**Проверьте:**
- Отображается название тега
- Хлебные крошки: Главная / Теги / [Название тега]
- Отображаются все новости с этим тегом
- Работает заглушка для битых изображений
- Если у тега нет новостей - показывается сообщение "Статьи с этим тегом не найдены"

### 4. Привязка тегов к новостям
1. Откройте любую статью в админке: http://localhost:3000/admin/collections/articles
2. Найдите поле "tags" (Теги)
3. Добавьте несколько тегов к статье
4. Сохраните
5. Проверьте, что статья отображается на страницах этих тегов

## 📝 Примеры URL:

- Список тегов: http://localhost:4321/tags/
- Тег "Мода": http://localhost:4321/tags/moda
- Тег "Красота": http://localhost:4321/tags/krasota

## 🔧 Команды для отладки:

```bash
# Проверить логи CMS
docker compose logs cms

# Проверить логи Web
docker compose logs web

# Перезапустить все контейнеры
docker compose restart

# Выполнить миграцию заново (если нужно)
docker compose exec cms npm run migrate

# Перегенерировать типы
docker compose exec cms npm run generate:types
```

## 📊 База данных:

Проверить данные в таблице tags можно через psql:
```bash
docker compose exec postgres psql -U payload -d payload -c "SELECT id, name, slug, meta_title FROM tags;"
```

## ⚠️ Возможные проблемы:

1. **CMS показывает "unhealthy"** - это нормально в первые 30 секунд после запуска
2. **404 на страницах тегов** - убедитесь, что web контейнер пересобран и перезапущен
3. **Нет slug у существующих тегов** - запустите миграцию: `docker compose exec cms npm run migrate`

## ✨ Особенности реализации:

- SEO-оптимизация с автогенерацией meta-тегов
- Единый дизайн с остальными страницами сайта
- Поддержка заглушек для изображений
- Хлебные крошки для навигации
